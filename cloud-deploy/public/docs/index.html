ZteAPI 接入文档

把客户端的 Base URL 改成 https://Zteapi.com/v1，填入你在 Sub2API 创建的 API Key，即可调用 GPT OAuth 或 NVIDIA adapter 模型。

Base URL

https://Zteapi.com/v1

认证方式

Authorization: Bearer YOUR_KEY

推荐模型

gpt-5.4 文本，gpt-image-2 生图

快速开始

场景填写内容
GPT / OpenAI OAuthAPI Key 使用绑定 GPT 组的 sk-...；文本先用 gpt-5.4，图片用 gpt-image-2。
NVIDIA adapterAPI Key 使用绑定 NVIDIA 组的 sk-sub2api-nvidia-...；模型先用 qwen3-next-80b。
协议文本优先使用 /v1/responses；老客户端可用 /v1/chat/completions。图片使用 /v1/images/generations。
不要把真实 API Key 写进公开代码、截图、网页或聊天记录。本文档只展示占位符。

Codex Desktop / Codex CLI

Codex 配置文件通常在 %USERPROFILE%\.codex\config.toml 或 ~/.codex/config.toml。推荐定义两个 provider，再用 profile 切换。

1. 设置环境变量

Windows PowerShell
[Environment]::SetEnvironmentVariable("ZTEAPI_GPT_KEY", "sk-your-gpt-sub2api-key", "User")
[Environment]::SetEnvironmentVariable("ZTEAPI_NVIDIA_KEY", "sk-your-nvidia-sub2api-key", "User")

2. 写入 config.toml

config.toml
model = "gpt-5.4"
model_provider = "zteapi_gpt"
model_reasoning_effort = "medium"
disable_response_storage = true

[model_providers.zteapi_gpt]
name = "ZteAPI GPT OAuth"
base_url = "https://Zteapi.com/v1"
wire_api = "responses"
env_key = "ZTEAPI_GPT_KEY"
env_key_instructions = "Set ZTEAPI_GPT_KEY to your Sub2API GPT user key."

[model_providers.zteapi_nvidia]
name = "ZteAPI NVIDIA"
base_url = "https://Zteapi.com/v1"
wire_api = "responses"
env_key = "ZTEAPI_NVIDIA_KEY"
env_key_instructions = "Set ZTEAPI_NVIDIA_KEY to your Sub2API NVIDIA user key."
stream_idle_timeout_ms = 600000

[profiles.zteapi-gpt]
model = "gpt-5.4"
model_provider = "zteapi_gpt"
model_reasoning_effort = "medium"

[profiles.zteapi-nvidia]
model = "qwen3-next-80b"
model_provider = "zteapi_nvidia"
model_reasoning_effort = "medium"

3. 启动指定 profile

CLI
codex -p zteapi-gpt
codex -p zteapi-nvidia
实际做代码 agent 工作时优先用 GPT profile。NVIDIA profile 更适合普通文本、模型备用或轻量测试。

配置生成器

选择 key 类型后生成最小 Codex provider 片段。密钥只在你的浏览器里处理，不会上传。

生成结果
点击“生成”。

HTTP 调用

查看模型

curl
curl https://Zteapi.com/v1/models \
  -H "Authorization: Bearer YOUR_SUB2API_KEY"

Responses API

GPT
curl https://Zteapi.com/v1/responses \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_GPT_SUB2API_KEY" \
  -d '{
    "model": "gpt-5.4",
    "input": "只回复 OK",
    "store": false,
    "max_output_tokens": 16
  }'
NVIDIA
curl https://Zteapi.com/v1/responses \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_NVIDIA_SUB2API_KEY" \
  -d '{
    "model": "qwen3-next-80b",
    "input": "Reply exactly: OK",
    "max_output_tokens": 16
  }'

图片生成

gpt-image-2 使用 GPT / OpenAI OAuth key，并且必须走 Images API。不要把图片模型发到普通聊天或 Responses 接口。

curl
curl https://Zteapi.com/v1/images/generations \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_GPT_SUB2API_KEY" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "一只橘猫坐在未来城市窗边，电影感光线",
    "size": "1024x1024"
  }'
返回结果里的 data[0].b64_json 是图片 base64 内容。NVIDIA key 不用于图片生成；如果用错 key 或接口，先检查 /v1/models 里当前 key 能看到哪些模型。

Python / Node SDK

Python 文本
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["ZTEAPI_GPT_KEY"],
    base_url="https://Zteapi.com/v1",
)

response = client.responses.create(
    model="gpt-5.4",
    input="只回复 OK",
    store=False,
)

print(response.output_text)
Node.js 文本
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.ZTEAPI_GPT_KEY,

});

const response = await client.responses.create({
  model: "gpt-5.4",
  input: "只回复 OK",
  store: false,
});

console.log(response.output_text);

模型

GPT OAuth 模型以后台可用模型为准，图片生成模型为 gpt-image-2。NVIDIA adapter 当前开放的是文本模型：

qwen3-next-80b qwen3-coder-480b llama-3.3-70b nemotron-super-49b kimi-k2.6 glm-5.1 deepseekv4-pro

用量与扣费

真实生成请求完成后，Sub2API 会记录 token 消耗。普通用户看自己的 Usage / 用量页面；管理员看 Admin / Usage Logs 或仪表盘聚合数据。

调用 /v1/models 通常不会产生模型 token 消耗。要验证仪表盘变化，请调用 /v1/responses 或 /v1/chat/completions。
图片生成成功后，管理员应在 Usage Logs 里看到 requested_model=gpt-image-2，并检查 image_output_tokens、total_tokens 和 total_cost。
如果 NVIDIA 模型能看到 input_tokens 和 output_tokens，但 total_cost 是 0，说明该模型在 Sub2API 定价表里还没配置价格或倍率；这不影响调用，但会影响成本统计。

充值与余额

如果请求返回 INSUFFICIENT_BALANCE，说明该用户余额不足，和本地是否开启 VPN 没有关系。登录后进入 充值页面 完成充值，余额到账后即可继续调用。

支付成功后系统会通过支付服务商 webhook 验签自动入账。不要把“支付成功跳转页”当作到账依据。
如果充值页面暂时没有支付方式，说明管理员还没有启用真实收款通道；请联系管理员配置 EasyPay、支付宝、微信或 Stripe。

常见错误

401 / invalid key

key 复制错误、已删除或不属于当前站点。

403 unsupported country

OAuth 授权出口地区不被 OpenAI 支持。

503 unavailable

该组没有可用账号，或模型没有开放。

model not found

模型名写错，先用 /v1/models 查看。

Unsupported parameter

移除报错中的参数，例如部分模型不支持温度。

图片接口错误

gpt-image-2 要走 /v1/images/generations。

Codex 仍走旧配置

重启终端或 Codex Desktop，确认 profile。
