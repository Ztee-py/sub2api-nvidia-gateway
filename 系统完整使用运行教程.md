# Sub2API NVIDIA 中转站完整使用与运行教程

本文档对应当前项目：`sub2api-https-build-nvidia-com-sub2api`。

目标系统：

```text
用户 / 客户端
  -> https://zteapi.com
  -> Caddy HTTPS 反向代理
  -> Sub2API
  -> nvidia-adapter
  -> NVIDIA NIM / build.nvidia.com API
```

当前已经完成：

- `zteapi.com` HTTPS 访问正常。
- Sub2API 后台可登录。
- NVIDIA 10 个 `nvapi-...` key 已恢复并全部探测通过。
- Adapter 已增加每 key 并发保护、排队等待、请求体大小限制。
- Adapter 已兼容 Sub2API 将 `/v1/chat/completions` 转发为 `/v1/responses stream=true` 的行为，会自动合成 Responses SSE `response.completed` 终止事件。
- Caddy 已增加 8MB 请求体限制，移除过早的 HSTS preload。
- 已提供服务器本地备份与巡检脚本。
- 已实测公网 `/v1/models`、`/v1/responses`、`/v1/chat/completions` 均可用，并能在 Sub2API 中产生调用数据。

## 3. 本地运行方式

本地运行主要用于调试 Adapter，不建议直接公网开放。

### 3.1 准备 `.env`

在项目根目录创建 `.env`，参考 `.env.example`：

```text
NVIDIA_API_KEYS=nvapi-xxx,nvapi-yyy
ADMIN_TOKEN=replace-with-admin-token
BIND_HOST=127.0.0.1
PORT=8000
REQUEST_TIMEOUT_SECONDS=180
KEY_COOLDOWN_SECONDS=90
MAX_RETRIES=10
KEY_MAX_IN_FLIGHT=1
KEY_QUEUE_WAIT_SECONDS=30
MAX_REQUEST_BODY_BYTES=8388608
```

### 3.2 检查配置

```powershell
python .\server.py --check-config
```

### 3.3 启动服务

```powershell
python .\server.py
```

打开本地仪表盘：

```text
http://127.0.0.1:8000/dashboard?token=你的_ADMIN_TOKEN
```

### 3.4 创建本地用户 token

```powershell
python .\server.py --create-user alice --quota 1000000 --note "local-test"
```

返回的 `sk-...` 只显示一次。

### 3.5 本地调用示例

```powershell
$key = "sk-你的用户token"

$body = @{
  model = "qwen3-next-80b"
  messages = @(
    @{ role = "user"; content = "Reply exactly: OK" }
  )
  max_tokens = 8
  temperature = 0
} | ConvertTo-Json -Depth 10 -Compress

curl.exe -i "http://127.0.0.1:8000/v1/chat/completions" `
  -H "Authorization: Bearer $key" `
  -H "Content-Type: application/json" `
  --data-raw $body
```

## 6. 客户端调用教程

### 6.1 curl 调用

```bash
curl "https://zteapi.com/v1/chat/completions" \
  -H "Authorization: Bearer 用户的sk" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen3-next-80b",
    "messages": [
      {"role": "user", "content": "Reply exactly: OK"}
    ],
    "max_tokens": 8,
    "temperature": 0
  }'
```

成功响应示例：

```json
{
  "id": "chatcmpl-a783435254cac103",
  "object": "chat.completion",
  "model": "qwen3-next-80b",
  "choices": [
    {
      "message": {
        "role": "assistant",
        "content": "OK"
      }
    }
  ],
  "usage": {
    "prompt_tokens": 14,
    "completion_tokens": 3,
    "total_tokens": 17
  }
}
```

说明：当前 Sub2API 的 chat 路由可能返回 `content-type: text/event-stream`，但响应体是完整 JSON。常见客户端可以正常读取；如果遇到特别严格的 SDK，再按具体 SDK 做兼容。

### 6.1.1 Responses API 调用

Sub2API 的 `/v1/responses` 入口也已验证可用：

```bash
curl "https://zteapi.com/v1/responses" \
  -H "Authorization: Bearer 用户的sk" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen3-next-80b",
    "input": "Reply exactly: SYSTEM_OK",
    "max_output_tokens": 12
  }'
```

成功时会返回：

```text
SYSTEM_OK
usage.total_tokens: 28
```

```bash
docker compose exec -T nvidia-adapter python /app/probe_upstream.py --model qwen3-next-80b --timeout 45
```

```bash
docker compose exec -T nvidia-adapter python /app/probe_upstream.py --model qwen3-next-80b --all-keys --timeout 45
```

探测代码模型：

```bash
docker compose exec -T nvidia-adapter python /app/probe_upstream.py --model qwen3-coder-480b --all-keys --timeout 90
```

如果出现：

```text

```

通常说明 key 无效或账号权限问题。

如果出现：

```text

```

通常是上游模型拥挤或该模型响应慢。可以提高 timeout，或暂时下架该模型。

## 9. 备份与恢复

### 9.1 服务器本地备份

不会下载到你的电脑：

```bash
cd /opt/sub2api-nvidia/cloud-deploy
./scripts/backup.sh
```

备份文件会保存到：

```text
cloud-deploy/backups/
```

包括：

```text
sub2api-postgres-*.sql.gz
sub2api-files-*.tar.gz
```

查看备份大小：

```bash
du -sh backups
ls -lh backups | tail -20
```

保留天数由 `.env` 控制：

```text
BACKUP_RETENTION_DAYS=7
```

### 9.2 升级前手动备份

```bash
cd /opt/sub2api-nvidia/cloud-deploy

stamp="$(date +%Y%m%d-%H%M%S)"
mkdir -p backups
source ./.env

docker compose exec -T postgres pg_dump -U "${POSTGRES_USER:-sub2api}" "${POSTGRES_DB:-sub2api}" | gzip -9 > "backups/pre-upgrade-postgres-${stamp}.sql.gz"

files=(.env docker-compose.yml Caddyfile data adapter_data)
for path in caddy_data caddy_config redis_data; do
  if [ -e "$path" ]; then
    files+=("$path")
  fi
done

tar czf "backups/pre-upgrade-files-${stamp}.tar.gz" "${files[@]}"
ls -lh backups | tail -20
```

### 9.3 恢复思路

恢复前先停止服务：

```bash
cd /opt/sub2api-nvidia/cloud-deploy
docker compose down
```

恢复文件备份：

```bash
tar xzf backups/sub2api-files-时间.tar.gz -C /opt/sub2api-nvidia/cloud-deploy
```

启动数据库后恢复 SQL：

```bash
docker compose up -d postgres redis
sleep 15
gunzip -c backups/sub2api-postgres-时间.sql.gz | docker compose exec -T postgres psql -U "${POSTGRES_USER:-sub2api}" "${POSTGRES_DB:-sub2api}"
docker compose up -d
```

恢复操作有破坏性，真实执行前建议再做一次当前状态备份。

## 11. 安全建议

必须做：

```text
1. 不公开 .env、NVIDIA key、Sub2API 管理员密码。
2. 已经暴露过的用户 sk 要删除重建。
3. 修改服务器 root 密码。
4. 后续改成 SSH key 登录，逐步禁用 root 密码登录。
5. 安全组和 UFW 只开放 22、80、443。
6. 不要公开 8000、8080、5432、6379。
7. Sub2API 后台启用 2FA。
8. 给用户设置并发、RPM、余额和模型权限。
```

UFW 基础命令：

```bash
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw enable
ufw status
```

如果以后使用 CDN，可以进一步只允许 CDN 回源 IP 访问 80/443。

## 12. 故障排查

### 12.1 访问网站打不开

检查：

```bash

```

### 12.2 HTTPS 证书失败

检查：

```bash

```

确认：

```text
域名解析到服务器公网 IP
80/443 已开放
Caddyfile PUBLIC_DOMAIN 正确
```

### 12.3 Sub2API 健康异常

```bash
docker compose logs --tail=120 sub2api
docker compose ps
```

常见原因：

```text
PostgreSQL 未健康
Redis 未健康
.env 密钥格式错误
TOTP_ENCRYPTION_KEY 不是正确格式
```

### 12.4 Adapter 异常

```bash
docker compose logs --tail=120 nvidia-adapter
docker compose exec -T nvidia-adapter python /app/server.py --check-config
```

常见原因：

```text
NVIDIA_API_KEYS 为空
ADAPTER_CLIENT_TOKEN 不一致
模型别名未加入
上游 NVIDIA 超时
```

### 12.5 Chat Completions 返回 400 / 502

如果 `/v1/chat/completions` 返回：

```text
400 Failed to parse request body
```

先确认请求体是标准 JSON，并确认 `Content-Type: application/json`。

如果日志里出现：

```text
Responses streaming is handled by the Sub2API front gateway
```

或：

```text
Upstream stream ended without a terminal response event
```

说明 Sub2API 将 Chat Completions 转发成了 Responses 流式上游调用，adapter 版本过旧。需要上传最新版 `server.py` 并重建 adapter：

```bash
cd /opt/sub2api-nvidia/cloud-deploy
docker compose build nvidia-adapter
docker compose up -d nvidia-adapter
sleep 20
docker compose ps
```

然后测试：

```bash
TOKEN='你的Sub2API用户APIKey'

curl -i --max-time 120 'https://zteapi.com/v1/chat/completions' \
  -H "Authorization: Bearer ${TOKEN}" \
  -H 'Content-Type: application/json' \
  --data '{"model":"qwen3-next-80b","messages":[{"role":"user","content":"Reply exactly: CHAT_OK"}],"max_tokens":12,"temperature":0}'
```

成功时应看到：

```text
HTTP/2 200
CHAT_OK
usage.total_tokens: 17
```

### 12.6 模型 404 / 410

说明模型 ID 不存在、账号无权限，或 NVIDIA 已下线模型。

已知：

```text
qwen/qwen2.5-coder-32b-instruct 已在 2026-05-12 下线
```

### 12.7 429 / 超时

可能是：

```text
用户并发太高
上游模型拥挤
NVIDIA key 达到平台限制
```

处理：

```text
降低用户并发/RPM
默认切到 qwen3-next-80b 或 llama-3.3-70b
暂时下架慢模型
```

### 12.8 health-check.sh 报 `.env: command not found`

通常是 `.env` 有 UTF-8 BOM：

